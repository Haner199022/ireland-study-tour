The Three Core AI Modules · 三大 AI 模块 · 围绕一口钟

A bell becomes a 3D model, a sound, an interaction — and a thing you hold.

三维IP→音乐→交互

Three modules, one chain, one bell. Day 1 turns the bronze bell into an faithful 3D model of a single chime bell and a full cultural-creative merch set you can print and hold; Day 2 gives that bell a theme song with AI music, anchored by the real recording; Day 3 turns it into an interactive page — built by describing it to Gemini — that loads the 3D bell and the music. Each module's output is the next one's raw material.

三个模块、一条线索、一口钟。Day 1 把一只青铜钟数字复刻成一个 3D 编钟（单钟）、并做成一整套文创周边（可打印、可拿在手里）； Day 2 用 AI 音乐给编钟配一首主题曲、用真实钟声锚定；Day 3 用自然语言对 Gemini 描述， 做成载入 3D 编钟与音乐的可交互页面。每个模块的产出，都是下一个模块的原料。

Day 1 · 3D & 文创 · 三维编钟与文创周边

三维编钟与文创周边3D Bianzhong & a Cultural-Creative Merch Set — from a bell to a shelf of products

The chain starts with a thing you can hold. Students digitally recreate a single bronze chime bell (one bell, not the full rack) as a real 3D model with text/image-to-3D AI (no modelling skills assumed), render a turntable, then design a complete 文创 (cultural-creative) merch set around it — a desk ornament, a blind box, a fridge magnet, a bookmark and more — and 3D-print one piece. The exported model is handed to Day 3, where the bell spins and reacts to the team's music.
这条线从「一个能拿在手里的东西」开始。学员用「文/图生 3D」把一只青铜编钟（单钟，而非整架）数字复刻成一个真正的 3D 模型（不预设建模基础），渲染转盘，再围绕它设计一整套文创周边——摆件、盲盒、冰箱贴、 书签……并打印一件实体。导出的模型交给 Day 3——让编钟随本组音乐旋转、共振。

当天产出 / Deliverable — 1 个 3D 编钟（单钟）模型（.glb）+ 360° 转盘渲染 + 一整套编钟文创周边（≥4 件样机：摆件 / 盲盒 / 冰箱贴 / 书签…）+ 1 件实体打印 / .stl + 文创系列 one-pager（导出的 .glb 供 Day 3 复用）

3D 编钟 & 文创 Workshop · 5 道循序练习
Exercise 1

Know the Bell先把编钟看清楚

15 min

目标 — 复刻前先认识形制。不碰 AI：挑一只真实编钟（单钟），记录关键特征（钮 · 枚 · 鼓部 · 纹饰 · 铭文 · 青铜锈色），收集多角度参考图；写一句「文创定位」（给谁 · 什么场景 · 什么调性）。

可选 · 即梦 / 豆包·图像 生成参考图
A clean reference sheet of a SINGLE ancient Chinese bronze chime bell
(one bell, not the full rack): accurate proportions, front and side views, the
bronze-green patina, the raised studs (mei) and inscriptions visible.
Neutral studio background, even lighting, photographic reference style.
步骤 · Steps
  1. 找 1–2 张博物馆单只编钟（不是整架）的清晰正面照，作主参考。
  2. 对照片列特征：钮 · 枚（凸点）· 鼓部 · 纹饰 · 铭文 · 青铜锈色。
  3. 写一句「文创定位」：给谁 · 什么场景 · 什么调性。
  4. （可选）用即梦按上方 prompt 生成一张干净正侧参考补充。
产出 / Deliverable
编钟特征清单 + 多角度参考图 + 一句文创定位。
讨论 / Discussion
  1. 编钟最有辨识度的 3 个特征是什么？
  2. 哪些细节拿来做文创最好用？
  3. 你的文创给谁、放在什么场景？
✦
教学点 — 文创不是把文物贴在杯子上——先读懂形制与气质，才知道留什么、舍什么。
Exercise 2

One Photo → A 3D Bell一张照片出 3D 编钟

25 min

目标 — 体验「图生 3D」：参考图 → 可旋转的 3D 编钟网格。把 Ex.1 的正面参考上传到混元3D / Tripo，先单图生成，背面穿帮再补三视图。

Hunyuan3D / Tripo — image-to-3D
Upload: your front-view single-bell reference image.
Mode: single image → 3D.  Symmetry: on.  Generate texture: on.
Polycount: medium.  Then preview the turntable.
If the curved bell body or rim looks wrong, switch to multi-view
and upload front + side + back.
步骤 · Steps
  1. 打开混元3D / Tripo，选「单图生 3D」。
  2. 上传 Ex.1 的单钟正面参考；开「对称」「生成贴图」，polycount 选 medium。
  3. 生成后转一圈，重点看钟体弧面 · 口部 · 钮。
  4. 穿帮就切「多视图」，补正 / 侧 / 背三张重生成。
产出 / Deliverable
1 个可在浏览器旋转的 3D 编钟模型 + 3 个角度截图。
讨论 / Discussion
  1. 钟体弧面 / 口部 / 钮 复刻准吗？哪里穿帮？
  2. 模型和真实编钟差在哪？
  3. 给一张图 vs 给三视图，差别有多大？
✦
教学点 — 图生 3D 是「猜立体」：参考越多，AI 猜得越准。三视图 + 比例参照 > 单图。
Exercise 3 · wow point

Clean It & Spin It修一下，转起来

30 min

目标 — 学会看 AI 网格的毛病、最小化处理，导出干净 .glb，做一个 360° 转盘渲染。不必会建模——会「看出哪不对、并描述出来」即可。

网格糊 / 多余面use the tool's "remesh / simplify / auto-retopo"
比例不对rescale, or regenerate with a clearer view
悬空 / 站不稳add a small round base or stand under the bell
颜色发灰re-bake the bronze texture, or tweak after export
步骤 · Steps
  1. 用工具内置 remesh / simplify 清掉多余面与糊面。
  2. 检查比例与站立，必要时加一个小底座 / 支架。
  3. 重新烘焙青铜贴图，或导出后再调色。
  4. 导出 model.glb（交 Day 3）。
  5. 录一段 10–15s 的 360° 转盘视频。
产出 / Deliverable
model.glb（交 Day 3）+ 360° 转盘视频 turntable.mp4（10–15s）。
讨论 / Discussion
  1. 哪个毛病最难修？
  2. 简化网格后「丢」了什么细节？值不值？
  3. 转起来之后，编钟比静态图更有质感吗？
✦
埋点 · 关键 — 这个 model.glb 就是 Day 3 交互可视化的主角——到时候让编钟随你的音乐旋转、共振。别丢。
Exercise 4

The 文创 Shelf做一整套文创周边

40 min

目标 — 把 3D 编钟变成一整套文创周边。选至少 4 种载体，统一一套视觉系统（配色 · 纹样 · 系列 logo），做贴图样机。

桌面摆件a miniature bell on a wood / acrylic base
盲盒a bell-series figure + a packaging box
冰箱贴 / 徽章bell silhouette + the bronze pattern
书签 / 胶带the bell's ornament flattened into a graphic
香插 / 茶宠a bell-shaped functional desk object
帆布袋 / 明信片a repeatable bell pattern
即梦 / Canva — product set mockup
A coordinated cultural-creative product set based on a bronze bianzhong:
a miniature bell desk ornament on a wood base; a blind-box package
(top logo "编钟 · 礼乐"); a fridge magnet; a bookmark. One visual system —
bronze-green palette + the bell's stud/inscription pattern. Studio
product photos, soft shadow, museum-shop e-commerce style.
步骤 · Steps
  1. 先定视觉系统：主色（青铜绿 + 暖金）· 一个纹样（取自枚 / 铭文）· 系列名 + logo。
  2. 选 ≥4 种载体（摆件 / 盲盒 / 冰箱贴 / 书签 / 香插 / 帆布袋…）。
  3. 用即梦 / Canva 按上方 prompt 逐件出贴图样机，保持同一视觉系统。
  4. 拼一张「系列合影」，配系列名与一句 slogan。
产出 / Deliverable
≥4 件文创周边样机 + 1 张「系列合影」+ 系列名 & 一句 slogan。
讨论 / Discussion
  1. 这套周边「成系列」吗？靠什么统一（色 / 纹 / 形）？
  2. 哪件最想买？哪件最有「博物馆文创」味儿？
  3. 还能延展出什么意想不到的单品？
✦
教学点 — 文创的关键是「成套的系统」：统一的视觉语言，让单品变成一个系列。
Exercise 5 · 高潮

Make It Real & Hand It Over打印实体 + 交接给交互课

30 min

目标 — 导出可打印的 .stl，切片并（轮流）打印一件（如编钟摆件）；同时把 .glb 正式交接给 Day 3 交互课。

A · 有打印机导出 STL → 切片（加底座 / 支撑，填充 10–20%）→ 轮流打印一件编钟摆件，课后取
B · 无打印机导出 STL + 高质量渲染图，组成「可打印包」；附在线打印服务报价截图
步骤 · Steps
  1. 导出 model.stl。
  2. 切片（Bambu / Cura）：加底座 + 支撑，填充 10–20%，预览无悬空。
  3. 轮流打印一件编钟摆件（课后取）；无打印机则交「STL + 渲染 + 报价截图」。
  4. 把 model.glb 正式交给交互组（Day 3）。
  5. 汇总成文创系列 one-pager（单钟 + 整套周边 + 团队）。
产出 / Deliverable
✅ model.stl · ✅ model.glb（交 Day 3）· ✅ 实体打印件或渲染图 · ✅ 文创系列 one-pager（编钟 + 整套周边 + 团队）。
讨论 / Discussion
  1. 实体摆件拿在手里 vs 屏幕上，感觉差在哪？
  2. 若打印失败，问题在模型还是切片？
  3. 这套文创能进博物馆礼品店吗？还差什么？
✦
收尾 · 埋点 — 你完整走了一遍博物馆文创的生产线：读形 → 复刻 3D → 整套周边 → 实体。把 model.glb 交给交互组——编钟即将随音乐起舞（Day 3）。

AI Music Workshop · 5 道循序练习
Exercise 1

Listen Like an AI像 AI 一样听歌

15 min

目标 — 学会反向拆解一首歌为 AI 可消化的 prompt 元素。教师播 3 段参考音乐（30s/段，各 2 遍），学员仅听、禁止搜歌名。

参考 1编钟独奏（如曾侯乙编钟复刻演奏）
参考 2古筝 + 笛子的现代国风（如《典籍里的中国》主题音乐）
参考 3中西融合电子
步骤 · Steps
  1. 每段闭眼听 2 遍，禁止搜歌名。
  2. 每段填 5 列：Genre · Instruments · Mood · Tempo · One word。
  3. 把「不喜欢的地方」翻成一个可避免的 prompt 词。
  4. 用英文写下三段共同的「中国感」来源。
产出 / Deliverable
每段填 5 列表：Genre · Instruments · Mood · Tempo · One word.
讨论 / Discussion

Exercise 4

Style as Culture风格即文化

35 min

目标 — 把视觉做成真水墨——渗墨、飞白、留白、泼墨，而不是「灰色几何形」。继续迭代，加入下面至少 3 项。

Apply at least 3 of these
Restyle it as REAL Chinese ink-wash (水墨 / sumi-e), not "grey shapes":
1. Canvas = warm rice-paper cream (#FAF6F0) with a faint fibrous grain
   and lots of empty space (留白 negative space).
2. Brush strokes, NOT geometric shapes: tapered ends, varied width, wet
   edges that BLEED softly into the paper (feathered alpha).
3. Ink in tonal washes (dark -> pale, 墨分五色) + dry-brush "flying-white"
   streaks (飞白) on fast strokes; ONE restrained accent only — a
   cinnabar-red seal 印章 OR jade #5B8C7E.
4. On a strong BEAT, throw a splashed-ink burst (泼墨): droplets scatter,
   then DIFFUSE and dry into the paper over ~1s.
5. The bass pulse should look like fresh ink BLOOMING on wet paper —
   not a circle scaling.
6. A single faint vertical centre line (Beijing Central Axis), a Chinese
   serif (Noto Serif SC), and a small red seal in one corner.
Reference look: Sesshu's "splashed-ink" (haboku) landscape.
步骤 · Steps
  1. 继续迭代，套用上方 ≥3 项水墨 / 中国风处理。
  2. （可选）用 <model-viewer> 放入 Day 1 的 3D 编钟，让它随低音转动。
  3. 删掉一个 AI 多给、不搭的元素。
  4. 保留完整 Prompt Log（所有迭代）。
产出 / Deliverable
视觉升级后的页面 + Prompt Log（保留所有迭代历史）。
讨论 / Discussion
  1. 这些视觉元素真的「中国」吗？还是只是符号？
  2. 你删掉了哪个 AI 给的元素？为什么？
  3. 不懂中国文化的观众能感觉到「水墨」吗？
✦
教学点 — 水墨的关键不是「黑白」，而是墨分五色 + 留白 + 笔触动态；印章 / 中轴是点睛，不是堆砌。

常见报错与急救 · Common failures & fixes（对照三案例真实代码）
点了播放没声音 Silent on playAutoplay policy — create/resume the AudioContext inside the play / ENTER click; call resume() whenever state==='suspended'.
麦克风/摄像头被拒 Mic/cam deniedgetUserMedia needs HTTPS / localhost, not file://; try/catch and reset the toggle on failure.
3D 模型不出现 Model missing404 / wrong path / blocked CDN / file absent → keep a 2.2s timeout + 'error' listener → SVG fallback, or use the drag-to-load path.
跨域加载失败 .glb CORSHost the .glb same-origin (same Pages/Netlify site), or base64-embed a small one as a data-URI.
createMediaElementSource 报错Only once per <audio> — make the AudioContext + source a guarded singleton (if(audioCtx) return).
掉帧 / 手机卡 Frame dropsCap rings/particles, shrink the frame-diff canvas (40×30), reuse one Uint8Array, disable shake/shadowBlur on weak devices.
多钟齐响爆音 Bell crackleInsert a DynamicsCompressorNode before destination to tame overlapping partials.
AR 桌面无反应 AR on desktopWebXR / Quick-Look are phone-only over HTTPS; treat AR as an optional enhancement, never required for full marks.
🆘 Gemini 反复给坏结果 · escalation ladder — 同一问题试 3 次仍失败就换方法，别死磕：① 报错原样粘回 + return the full corrected HTML；② 缩小范围 Only change the audio part; keep the rest identical；③ 回滚到上一版能跑的 HTML，再只加一小步；④ 仍不行 → 叫老师 / 换更简单的交互 / 直接拿三案例之一二创。铁律：永远留一份「上一版能跑的 HTML」。
