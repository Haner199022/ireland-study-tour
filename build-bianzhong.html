Case B + C · Deep Build · 深拆可交互编钟

一步步做出「可交互编钟」Build the Playable Bianzhong — one prompt at a time

This is the full teardown of the playable 3D bianzhong page — orbit the bronze bells, then tap them to ring tuned notes. We rebuild it from an empty stage into a finished instrument in 7 steps, and every step is just one plain-English sentence you paste into Gemini. You never write code; you describe, you run, you check, you say the next sentence.

这是「可交互 3D 编钟」页面的完整拆解——旋转那架青铜钟，再点击钟体奏出调准的音。我们从一个空舞台开始，用 7 步把它做成一件成品乐器，而每一步都只是你粘进 Gemini 的一句大白话。你不写代码；你描述、你运行、你检查，然后说下一句。

动手前 · 三样东西Before you start — three things

  • 一个 Gemini — 打开 Google AI Studio 或 Gemini App 的 Canvas，登录即可。Open Gemini Canvas (AI Studio or the Gemini app) and sign in.
  • 一个 3D 模型 — Step 1 用课程托管好的 编钟.glb 网址；如果该网址暂时取不到，Step 2 会让你把自己的 .glb 拖进舞台。A 3D model — Step 1 uses the course-hosted 编钟.glb URL; if it isn't reachable yet, Step 2 lets you drag in your own .glb.
  • 一个现代浏览器 — 本例不强求 localhost（模型从网址加载、声音用 Web Audio，都能直接跑）；只有手机 AR 需要 https，且永远是加分项。A modern browser — no localhost needed here; only phone AR needs https, and AR is always a bonus.

每一步都跑同一个循环：描述 → 预览 → 验证 → 迭代。每次跑通，先存一份再往下改——「上一个能跑的版本」是你最好的安全网。

Every step runs the same loop: describe → preview → verify → iterate. Save each working version before you change it — your last working copy is your safety net.

7 步阶梯The 7-step ladder — empty stage to a playable instrument

Step 1 builds the whole shell; Steps 2–7 are all said into the same Gemini chat — each one adds one capability on top of the last. 第 1 步搭好骨架，第 2–7 步都在同一个对话里继续说，每句加一样新能力。

1

3D 舞台：载入编钟The 3D stage: load the bell

目标 — 一个能旋转、缩放、自转的 3D 编钟，配米纸+青铜的底子，外加一个 AR 按钮。Goal — a 3D bianzhong you can orbit, zoom and watch auto-rotate, on a rice-paper + bronze theme, with an AR button.

Prompt · 复制粘贴给 Gemini 
Build a single self-contained HTML file: an interactive 3D viewer for a
Chinese bianzhong (ancient bronze chime bells). No build step. Load
Google's model-viewer web component from unpkg, and load the model from
this URL:
  https://haner199022.github.io/ireland-study-tour/models/bianzhong.glb
Give it orbit + zoom + a gentle auto-rotate (resuming after a short idle
delay), soft shadows, neutral environment lighting, and an AR button.
Use a warm rice-paper (#FAF6F0) and bronze (#B08D57) visual theme with
bilingual 中文 + English labels. Keep it elegant, with lots of calm space.
✓ 验证 Verify
  • 钟出现了；能拖动旋转、滚轮缩放、自己缓缓转。The bell appears; you can orbit, zoom, and it slowly auto-rotates.
  • 手机上能看到 AR 按钮。An AR button shows on a phone.
底层揭秘 Under the hood
model-viewer 是一个网页标签，只要一个 .glb 文件，就同时给你 3D 查看 + AR，不用自己写 3D 引擎。model-viewer is one web component: a single .glb gives you 3D viewing + AR, with no 3D engine to write.
2

取不到也不崩：空状态 + 拖入自己的模型Never break: empty state + drag in your own model

目标 — 模型网址失败时，给一张友好的「放入你的模型」卡片，而不是空白；并能把自己的 .glb 拖进舞台即时预览。Goal — if the URL fails, show a friendly "add your model" card instead of a blank stage; and let me drag my own .glb in to preview instantly.

Prompt · 同一个对话，继续说 
In the same file, make it robust. If the model URL fails to load
(missing or offline), do NOT show a blank or broken stage — show a
friendly "add your bianzhong model" empty card over the stage instead.
And let me load my own model: I can drag a .glb or .gltf file onto the
stage, or click a button to pick one, and it loads and previews instantly.
✓ 验证 Verify
  • 没有模型时，出现「放入你的模型」卡片，页面不空、不报错。With no model, the "add your model" card shows — no blank, no crash.
  • 把一个 .glb 拖到舞台上，立刻显示。Drag a .glb onto the stage — it loads at once.
底层揭秘 Under the hood
model-viewer 会发出 load / error 事件：出错就显示空卡；拖进来的文件用一个临时 object URL 直接喂给它。model-viewer fires load / error events: on error, show the empty card; a dragged-in file is handed to it via a temporary object URL.
3

点一下，钟响：合成钟声Tap to ring: a synthesized bell

目标 — 点击模型就响起一声有金属质感、带长尾的钟声——不是电子「滴」声。Goal — tapping the model rings a metallic, long-decaying bell tone — not an electronic beep.

Prompt · 同一个对话，继续说 
Now make it play. When I tap anywhere on the model, ring a bell tone
synthesized with the Web Audio API. Don't use a single sine beep — stack
about six inharmonic partials (slightly detuned overtones) with a fast
attack and a long, roughly 3-second exponential decay, so it sounds like
real struck bronze rather than an electronic tone.
✓ 验证 Verify
  • 点钟体 → 响起一声金属感、余韵很长的钟声。Tap the bell → a metallic tone with a long ring-out.
  • 听起来像青铜，不像「滴」。It sounds like bronze, not a beep.
底层揭秘 Under the hood
真实的钟是「非谐」的——把几个略微失谐的泛音叠在一起，才有青铜的质感；单一正弦只会是干净的电子音。Real bells are inharmonic — stacking a few detuned overtones gives the bronze timbre; a single sine is just a clean electronic note.
4

音高随位置走 + 五声音阶Pitch by position + the pentatonic scale

目标 — 点得越靠左越低、越靠右越高，走中国五声音阶 宫商角徵羽；每次发声，屏幕中央闪出音名。Goal — left = low, right = high, on the 宫商角徵羽 pentatonic scale; each ring flashes the note name.

Prompt · 同一个对话，继续说 
Map the pitch to WHERE I tap, horizontally across the model: left = low,
right = high. Use a Chinese pentatonic scale — 宫商角徵羽
(gong-shang-jue-zhi-yu) — 10 notes spanning about two octaves:
G3 A3 C4 D4 E4 G4 A4 C5 D5 E5. Each time a note rings, flash its name
large over the stage.
✓ 验证 Verify
  • 点左边=低音，点右边=高音；音高沿五声音阶走。Tap left = low, right = high; pitch follows the pentatonic scale.
  • 每响一声，中央闪出对应音名。Each ring flashes its note name in the centre.
底层揭秘 Under the hood
从点击点取出 3D 命中坐标，把它的 X 在模型宽度上归一化（0→1）= 音的序号；取不到命中就退回用屏幕像素 X。Get the 3D hit point of the tap, normalise its X across the model width (0→1) → the note index; if there's no hit, fall back to the pixel X across the stage.
5

一排可点的音键An on-screen key pad

目标 — 舞台下方一排音键，点哪个响哪个并高亮；和点模型用的是同一套音。Goal — a row of note keys under the stage; click one to ring + light it; same notes as tapping the model.

Prompt · 同一个对话，继续说 
Add a row of clickable note keys below the stage — one key per note in
the scale, labelled with the note name. Clicking a key rings that note
and briefly lights the key. Tapping the model and clicking the keys must
use the exact same set of notes.
✓ 验证 Verify
  • 下方出现一排音键；点某个键 → 响对应音并亮一下。A row of keys appears; click one → it rings and lights briefly.
  • 键和点模型发的是同一套音，不跑调。Keys and model taps use the same notes — no drift.
底层揭秘 Under the hood
音键和模型点击读同一个「10 音数组」——一处定义、两处共用，是它们不跑调的原因。Keys and model taps both read one shared "10-note array" — one definition, used in two places, is why they stay in tune.
6

奏一段 + 涟漪与高亮反馈Run the scale + ripple & key-light feedback

目标 — 一个「奏一段」按钮顺次奏完全部音；每次发声，点击处冒出一圈涟漪、对应键亮一下。Goal — a "Run the scale" button plays all notes in order; each ring sends a ripple from the tap point and lights the matching key.

Prompt · 同一个对话，继续说 
Add a "奏一段 / Run the scale" button that plays all the notes in order,
low to high, with a short gap between each. And add visual feedback on
every ring: send a soft expanding ripple out from the point I tapped, and
light up the matching key on the key pad (keep the note-name flash you
already have).
✓ 验证 Verify
  • 点「奏一段」→ 从低到高顺次奏完。Click "Run the scale" → it plays low to high in order.
  • 每次发声，点击处冒出一圈涟漪，对应音键亮一下。Each ring sends a ripple from the tap point and lights the key.
底层揭秘 Under the hood
「奏一段」= 给每个音排一个小延时（约 0.23 秒一个）顺序触发；涟漪是在点击坐标处放一个会扩散、淡出的小圆。"Run the scale" schedules each note on a small delay (~0.23s apart); the ripple is a little circle placed at the tap point that expands and fades.
7

控件、收尾与上线Controls, polish & going live

目标 — 自动旋转开关 + 复位视角键；通页米纸+青铜、中英双语；导出一个完整文件。再把模型「上线」。Goal — an auto-rotate toggle + reset-view button; rice-paper + bronze, bilingual throughout; output one file. Then get your model online.

Prompt · 同一个对话，继续说 
Finish it. Add an auto-rotate on/off toggle and a "reset view" button that
re-centres the camera. Keep the whole page on the rice-paper-and-bronze
theme with bilingual 中文 + English labels throughout. Output one complete,
self-contained HTML file.
✓ 验证 Verify
  • 自动旋转开关能开关；复位键把视角拉回正面。The auto-rotate toggle works; reset pulls the camera back to front.
  • 通页米纸+青铜配色，标签中英双语。Rice-paper + bronze throughout; labels bilingual.
底层揭秘 Under the hood
autoRotate、cameraOrbit 都是 model-viewer 的可读写属性，开关/复位直接设它们即可。autoRotate and cameraOrbit are read/write properties on model-viewer — the toggle and reset just set them.

上线你的模型 — 「上传」其实就是：把 .glb 放到一个网页能用网址取到的地方。最省事的办法：把文件放进 website/models/ 推送到 GitHub Pages，得到一个公开网址填回 Step 1；或者把 .glb 以 base64 内嵌成 data-URI，单文件即可运行。详见 编钟页的「托管 + Gemini 流程」↗。Going live — "upload" just means putting the .glb where a page can fetch it by URL: drop it in website/models/ and push to GitHub Pages for a public URL (paste it back into Step 1), or base64-embed it as a data-URI for one self-contained file. See the bianzhong page's hosting flow ↗.
